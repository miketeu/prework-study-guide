Git

  • At the CLI
    • git status: checks what branch we are currently on.
    • git checkout -b (branch name): creates a new branch and switches to it.
    • git checkout features/add-(branchname): puts us in the correct branch.
    • git add -A: To stage changes.
    • git commit -m ("Add your own message here")
    • git pull origin main: Make sure local branch is up to date before pushing up the changes.
    • git push origin feature/add-(branch-name): To push up changes to the repository.
  • Navigate to GitHub:
    • Navigate to correct repository.
    • change branch to features/add-(branch-name)
    • Navigate to "Pull Requests" and open a new pull request. Make sure to select 'main' as the base branch and features/add-(branch-name) as the comparison branch.
    • Click 'Create Pull Request' to open PR.
    • Once on the pull request form, give your PR a title and a short description of the changes.
    • Click the "Merge pull request" button to merge your work into the 'main' branch.
